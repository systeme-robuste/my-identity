# ⚠️ KNOWN MISMATCHES — Drizzle schema vs SQL migration

> The Drizzle schemas in this folder are **placeholders** for Drizzle Kit to discover. The source of truth is `migrations/0001_initial.sql` (543 lines, 28 tables).

> The placeholder schemas will be **out of sync** with the SQL migration until Phase 1, when Drizzle Kit takes over (`drizzle-kit generate` → `drizzle-kit migrate`).

## When to fix

Phase 1, step 1 ("Infrastructure: Drizzle Kit setup"). Not blocking for M0 push but blocks the first real `db:generate` + `db:migrate`.

## How to fix

1. Read the entire SQL migration
2. For each of the 28 tables, verify (or write) the Drizzle schema
3. Run `pnpm --filter @my-identity/db generate` and check that the generated SQL is byte-equal to the existing migration
4. If not, fix the Drizzle schema (don't touch the SQL)
5. Commit the fix as `0001_realign.ts` (Drizzle will append, not overwrite)
